**Версия документа:** 1.0  
**Дата:** Июнь 2026  
**Платформа:** Telegram Bot API  
**Стек:** Python 3.11+, Aiogram 3, PostgreSQL, SQLAlchemy (async), ЮKassa / Telegram Stars

## 1. КОНЦЕПЦИЯ ПРОДУКТА

### 1.1. Идея
Пользователь хочет узнать, насколько лучше он может выглядеть без пластики и дорогих процедур. Бот проводит персонализированный разбор внешности и выдаёт отчёт.

### 1.2. Целевая аудитория
- Мужчины и женщины 18–50 лет
- Интересуются своей внешностью, но не готовы к радикальным мерам
- Хотят конкретных рекомендаций «что исправить»

### 1.3. Что получает пользователь после оплаты
1. **Текущий уровень реализации потенциала** (оценка в %)
2. **Сильные стороны внешности**
3. **Основные ошибки в образе**
4. **3 точки максимального роста** (что даст 80% результата)
5. **Прогноз результата после изменений**
6. **Персональный план действий** (пошагово)

## 2. USER FLOW (ПОЛНЫЙ ПУТЬ ПОЛЬЗОВАТЕЛЯ)

```
Старт → Приветствие → Имя → Возраст → Цель → Фото(1-3) 
→ Описание продукта → Ожидание оплаты → Оплата (Stars/ЮKassa) 
→ Ожидание результата → Получение результата → Exit / Промо
```

|-----|----------|---------------|

| 4 | Запрос возраста | Текст |

## 3. СТРУКТУРА БАЗЫ ДАННЫХ (PostgreSQL)

### 3.1. Таблица `users`

```sql
CREATE TABLE users (
    id              BIGSERIAL PRIMARY KEY,
    telegram_id     BIGINT UNIQUE NOT NULL,
    username        VARCHAR(255),
    first_name      VARCHAR(255),
    name            VARCHAR(255),        -- введённое имя
    age             INTEGER,
    goals           TEXT[],              -- массив целей
    photo_ids       TEXT[],              -- file_id Telegram
    status          VARCHAR(32) NOT NULL DEFAULT 'new',
        -- new | awaiting_payment | paid | in_progress | completed | cancelled
    payment_method  VARCHAR(32),         -- 'stars' | 'yukassa'
    payment_id      VARCHAR(255),        -- ID транзакции
    payment_amount  DECIMAL(10,2),
    result_text     TEXT,
    result_file_id  VARCHAR(255),        -- file_id готового отчёта
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);
```

### 3.2. Таблица `admin_settings`

```sql
CREATE TABLE admin_settings (
    id              SERIAL PRIMARY KEY,
    admin_chat_id   BIGINT NOT NULL,
    created_at      TIMESTAMP DEFAULT NOW()
);
```

### 3.3. Индексы

```sql
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_created ON users(created_at);
CREATE INDEX idx_users_telegram ON users(telegram_id);
```

## 5. FSM (FINITE STATE MACHINE) — СОСТОЯНИЯ

```python
class UserState(StatesGroup):
    name = State()              # Ожидание имени
    age = State()               # Ожидание возраста
    goals = State()             # Выбор целей
    photos = State()            # Ожидание фотографий
    confirm = State()           # Подтверждение данных
    payment_method = State()    # Выбор оплаты
    awaiting_payment = State()  # Ожидание оплаты
    paid = State()              # Оплачено, ожидание результата
```

## 8. ПЛАТЁЖНАЯ СИСТЕМА

### 8.1. Telegram Stars

- Используется `Bot.sendInvoice()` с параметром `currency='XTR'`
- Цена: 299 ⭐
- Подтверждение: хендлер `pre_checkout_query` → ответ `OK`
- Финализация: `message_successful_payment` → статус `paid`

**Формирование счёта:**
```python
await bot.send_invoice(
    chat_id=user_id,
    title="Персональный разбор внешности",
    description="Анализ внешности по 12 параметрам + план действий",
    payload=f"user_{user_id}_{int(time())}",
    provider_token="",  # пусто для Stars
    currency="XTR",
    prices=[LabeledPrice(label="Разбор внешности", amount=299)],
)
```

### 8.2. ЮKassa

- Провайдер-токен из `.env`: `YUKASSA_PROVIDER_TOKEN`
- Валюта: `RUB`
- Цена: `1199.00`
- Используется `Bot.sendInvoice()` с корректным `provider_token`
- Webhook уведомлений от ЮKassa (или polling)
- После подтверждения → статус `paid`

## 9. СИСТЕМА СТАТУСОВ

| Статус | Описание | Эмодзи |
|--------|----------|--------|
| `new` | Новая заявка, данные собраны | 🆕 |
| `awaiting_payment` | Пользователю отправлен счёт | ⏳ |
| `paid` | Оплата подтверждена | ✅ |

| `completed` | Результат отправлен | ✅ |
| `cancelled` | Отменено | ❌ |

**Переходы:**
```
new → awaiting_payment → paid → in_progress → completed
new → cancelled (в любой момент)
```

## 12. ТРЕБОВАНИЯ К РЕАЛИЗАЦИИ

### 12.1. Общие
- Асинхронный код на `asyncio` / `Aiogram 3`
- Поддержка `FSM` для многошаговых форм
- Graceful shutdown (disconnect bot + DB)

## 14. ЗАМЕТКИ ПО UX

1. **Никакого мусора** — после регистрации бот не шлёт спам
2. **Кнопка «Назад»** — на каждом шаге, где уместно
3. **Прогресс** — на этапе сбора фото показывать счётчик «1/3»
4. **Сообщения-заглушки** — если пользователь шлёт не то, бот вежливо просит ввести корректные данные
5. **Подтверждение** — перед оплатой обязательный экран «Проверь свои данные»
6. **Тишина в работе** — пока админ готовит результат, бот не беспокоит пользователя
